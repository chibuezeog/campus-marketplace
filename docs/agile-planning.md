# Agile Planning

## Project Overview

Campus Marketplace is a cloud-based product catalog service that allows customers to browse products and catalog managers to manage inventory.

This project is being developed using Agile principles and Scrum practices. Work is organized into a product backlog, prioritized into sprints, and tracked using a GitHub Kanban board.

---

## Product Goal

Develop a reliable product catalog service that supports product management, customer interaction, and cloud deployment.

---

## Product Backlog

The initial backlog includes the following features:

- Create a product
- Retrieve a product
- Update a product
- Delete a product
- Like a product
- Dislike a product
- List all products
- Query a subset of products
- Host the application in the cloud
- Automate cloud deployments

---

## Sprint 1 Goal

Deliver the core product management functionality by implementing the CRUD operations:

- Create Product
- Retrieve Product
- Update Product
- Delete Product

---

## Estimation

Story points were assigned using relative estimation to reflect the effort and complexity of each feature.

| Feature | Story Points |
|---------|-------------:|
| Create Product | 3 |
| Retrieve Product | 2 |
| Update Product | 3 |
| Delete Product | 2 |

---

## Acceptance Criteria

Each user story includes acceptance criteria written in Gherkin syntax using the Given–When–Then format to define testable requirements.

---

## Project Management

The project uses a GitHub Kanban board with the following workflow:

- Product Backlog
- Sprint Backlog
- In Progress
- Review
- Done

This workflow helps organize work and track progress throughout development.
