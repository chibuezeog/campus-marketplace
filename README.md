# Campus Marketplace


Campus Marketplace is my first DevOps portfolio project.


## Project Overview

Campus Marketplace is a cloud-based product catalog service designed to help customers browse products while allowing catalog managers to manage inventory. This project follows an Agile workflow from planning through development, testing, deployment, and maintenance.

Rather than being built all at once, this project evolves as new DevOps skills are learned and applied.

## Current Status

**Current Phase:** Agile Planning ✅

Completed:

* Product Backlog
* Sprint Backlog
* User Stories
* Gherkin Acceptance Criteria
* Story Point Estimation
* GitHub Kanban Board

## Planned Features

* Create a product
* Retrieve a product
* Update a product
* Delete a product
* Like a product
* Dislike a product
* List all products
* Query products using filters
* Cloud deployment
* Automated CI/CD pipeline

## Agile Workflow

This project is managed using GitHub Issues and a Kanban board with the following workflow:

* Product Backlog
* Sprint Backlog
* In Progress
* Review
* Done

Each feature begins as a user story, includes Gherkin acceptance criteria, and is estimated using story points.
